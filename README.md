# PurchaseService: Автоматизация сбора и комплаенс-анализа закупок Группы Сбербанк (223-ФЗ)

ETL-пайплайн и аналитический контур для мониторинга закупочной деятельности, выявления инфраструктурных зависимостей от макроэкономических факторов (курс USD, ключевая ставка ЦБ) и автоматического обнаружения комплаенс-рисков.

## Архитектура и ключевой стек технологий
* **Язык разработки:** Python 3.10+
* **Управление конфигурацией:** Централизованный `src/config.py` 
* **Сбор данных / Эмуляция:** `ftplib`, `pandas` (концептуальный клиент ЕИС)
* **Хранилище данных:** PostgreSQL (реляционная схема, пакетная вставка через `psycopg2.extras.execute_values`, индексирование)
* **Аналитический слой:** Jupyter Notebook, `pandas`, `sqlalchemy`, `matplotlib`, `seaborn`

---

##  Архитектура базы данных (DDL)

Проект реализует реляционную модель данных, оптимизированную для аналитических выборок:
* `dict_customers` — справочник дочерних структур.
* `dict_suppliers` — справочник контрагентов-поставщиков.
* `purchases` — таблица фактов, содержащая метрики лотов, временные метки и срезы макроэкономических показателей.

Для ускорения выборок по периодам на уровне СУБД развернуты B-Tree индексы:
* `idx_purchases_date` на поле `publish_date`
* `idx_purchases_category` на поле `category`

## Инженерное обоснование использования синтетических данных

В модуле `src/parse_eis.py` реализован гибридный подход: скрипт осуществляет **реальное подключение** к государственному FTP-серверу ЕИС (`ftp.zakupki.gov.ru`) для проверки сетевой доступности продакшен-источника, после чего переключается на встроенный генератор изолированного датасета.

### Преимущества такого подхода:

1. **Изоляция тестов и воспроизводимость** Полный архив XML-выгрузок ЕИС по закупкам Сбербанка за 2024–2025 гг. весит десятки гигабайт. Скачивание и обработка такого массива данных в рамках демонстрационного или тестового запуска нецелесообразны. Синтетический генератор за доли секунды создает компактный пулл данных, полностью пригодный для валидации пайплайна.
  
3. **Контроль тестовых сценариев** Реальные данные ЕИС хаотичны. Чтобы доказать работоспособность аналитического контура, в генератор были детерминированно заложены строгие математические паттерны:
   * **Проверка дедупликации:** Скрипт принудительно инжектирует ровно 15 избыточных полных дубликатов строк. Это позволяет на этапе ETL (`src/database.py`) валидировать работу метода `.drop_duplicates()`.
   * **Макроэкономическая привязка:** В данные заложена экспоненциальная зависимость стоимости ИТ-оборудования от волатильности курса USD во второй половине 2025 года для проверки корреляции Пирсона.
4. **Обход инфраструктурных ограничений ЕИС:** Государственный FTP-сервер ЕИС имеет жесткие лимиты на количество одновременных сессий и регулярно уходит в таймаут в часы нагрузки. Использование Mock-пайплайна гарантирует работу проекта без внешних сетевых зависимостей.

## Ключевые бизнес-выводы аналитического модуля

### 1. Макроэкономическое влияние 
* **Наблюдение:** Между совокупным объемом начальных цен  и курсом доллара США зафиксирована сильная прямая связь ($r \approx 0.74$). Связь с ключевой ставкой ЦБ значительно слабее.
* **Интерпретация:** Закупочный контур Группы Сбербанк критически импортозависим. Девальвация рубля автоматически удорожает сервера и софт, поставляемые по параллельному импорту, вынуждая заказчиков превентивно раздувать бюджеты НМЦ.
* **Рекомендация:** Внедрение казначейских инструментов валютного хеджирования при планировании ИТ-инфраструктуры на 2026 год.

### 2. Комплаенс-контроль и аномалии конкуренции
* **Наблюдение:** Поставщик *ООО «СберИнтеграция»* занимает доминирующее положение, выиграв 97 контрактов со снижением цены ровно `0.00%`. На уровне заказчиков аномалию подтверждает *АО «СберТех»*, имеющий средний процент экономии по всем торгам `0.00%` (в отличие от здоровых `~10%` у ПАО Сбербанк).
